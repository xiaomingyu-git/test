## 文档结构

### 任务文档标准

每个任务包含三个核心文档：

1. **`[task]-plan.md`** - 战略规划和总体设计
2. **`[task]-context.md`** - 技术上下文和实施细节
3. **`[task]-tasks.md`** - 详细任务清单和进度追踪

### 使用指南

#### 创建新任务
```bash
# 使用/dev-docs命令创建新任务
/dev-docs [任务描述]
```

#### 查看任务进度
1. 进入任务目录：`cd dev/active/[task-name]/`
2. 查看任务清单：`cat [task]-tasks.md`
3. 查看技术上下文：`cat [task]-context.md`
4. 查看总体计划：`cat [task]-plan.md`

#### 更新任务状态
- 在`[task]-tasks.md`中标记已完成任务 ✅
- 在`[task]-context.md`中记录技术决策和实施细节
- 更新进度百分比和时间统计

## 开发规范

### 任务命名规范
- 使用kebab-case命名：`task-name`
- 目录名与任务名保持一致
- 文件名格式：`[task-name]-[type].md`

### 文档更新规范
- 每次重要更新后修改"最后更新"时间
- 使用✅、🟡、🔴等emoji标记状态
- 保持实施细节的准确性和完整性

### 任务完成规范
- 所有子任务标记为 ✅
- 更新context.md包含最终实施状态
- 记录经验教训和最佳实践
- 移动到`dev/completed/`目录
